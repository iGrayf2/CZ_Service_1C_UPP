# План проекта

Главная цель: сервис интеграции 1С УПП с Честным Знаком.

## Принципы

1. 1С остается интерфейсом для пользователей.
2. Вся логика Честного Знака живет в Python-сервисе.
3. Python не пишет напрямую в базу 1С.
4. Состояния кодов маркировки хранятся в отдельной базе сервиса.
5. Все важные действия должны логироваться.

## Итерации

### v0.1 — связь 1С и Python

- FastAPI-приложение.
- Метод `GET /ping`.
- Проверка из браузера.
- Проверка из 1С.

### v0.2 — настройки и логирование

- Конфигурационный файл.
- Логи сервиса.
- Единый формат ошибок.

### v0.3 — база сервиса

- PostgreSQL.
- Таблицы кодов маркировки.
- Таблицы заданий печати.
- Журнал событий.

### v0.4 — Честный Знак

- Авторизация.
- Тестовый контур.
- Получение статусов.

### v0.5 — коды маркировки

- Заказ кодов.
- Хранение кодов.
- Резервирование кодов под заказ/упаковку.

### v0.6 — печать Godex

- Генерация этикетки.
- Отправка на сетевой Godex.
- Повторная печать.

### v1.0 — рабочая интеграция с УПП

- Команды из 1С.
- Ответы пользователю в 1С.
- Учет маркируемых и немаркируемых товаров.
